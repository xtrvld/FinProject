Описание работы
Данная программа представляет собой упрощённую реализацию системы контроля версий,
аналогичной Git. Система позволяет инициализировать репозиторий, сохранять состояния
файлов проекта, просматривать историю изменений, сравнивать версии файлов и работать
с ветками.

Язык программирования: C++
Среда разработки: Microsoft Visual Studio
Стандарт языка: C++17

Все служебные данные системы хранятся в скрытой директории .simple_vcs в корне проекта.

---

## Поддерживаемые команды

init
Инициализирует репозиторий. Создаёт директорию .simple_vcs, файл HEAD и начальную
ветку master.

commit -m "сообщение"
Сохраняет текущее состояние файлов проекта.
Файлы сохраняются как объекты по хэшу.
Каждый коммит содержит ссылку на дерево файлов и предыдущий коммит.

checkout <хэш | имя_ветки>
Восстанавливает состояние файлов проекта по указанному коммиту или ветке.
Рабочая директория полностью перезаписывается сохранённым состоянием.

diff
Показывает различия между версиями файлов.
Используется построчное сравнение на основе алгоритма LCS.
Добавленные строки помечаются '+', удалённые '-'.

branch
Отображает список веток.

branch <имя_ветки>
Создаёт новую ветку от текущего коммита.

branch -d <имя_ветки>
Удаляет указанную ветку (если она не является текущей).

log
Отображает историю коммитов текущей ветки.

log --oneline
Отображает историю коммитов в кратком формате.

## Инструкция по сборке

1. Открыть проект в Microsoft Visual Studio.
2. Выбрать стандарт C++17.
3. Выполнить Build → Build Solution.
4. После сборки будет создан исполняемый файл FinProject.exe.

---

## Инструкция по запуску

Открыть PowerShell или командную строку в папке с исполняемым файлом и выполнять команды
в следующем формате:

.\FinProject.exe init
.\FinProject.exe commit -m "First commit"
.\FinProject.exe log --oneline
.\FinProject.exe diff
.\FinProject.exe branch feature
.\FinProject.exe checkout master

---

## Принцип работы

Файлы проекта сохраняются как blob-объекты по хэшу SHA-1.
Состояние проекта описывается деревом файлов (tree-объект).
Коммиты содержат ссылку на tree-объект и предыдущий коммит.
Ветки реализованы как указатели на коммиты.
Текущая позиция в истории хранится в файле HEAD.
